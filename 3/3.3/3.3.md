# 3.3. Consolidación de Escenarios

[Volver al índice](../3.md)

Este documento presenta la consolidación de los 30 escenarios preliminares identificados en la fase de brainstorming, agrupándolos en 20 escenarios consolidados según sus atributos de calidad y características comunes. Esta consolidación facilita el análisis arquitectónico y la priorización de decisiones de diseño.

---

## Tabla de Escenarios Consolidados

| **ID Escenario** | **Atributo de Calidad** | **Estímulo** | **Respuesta** | **ID Escenario Preliminar** | **Sustento** |
|------------------|------------------------|--------------|---------------|----------------------------|--------------|
| ESC-01 | Rendimiento | 300 usuarios concurrentes generan listados con filtros avanzados, consultan usuarios iniciales y publican reseñas masivamente durante evaluaciones | El sistema procesa todas las solicitudes sin degradación, mostrando listados en <3s, carga inicial en <5s y publicaciones en <1.5s por reseña | ESCP-01, ESCP-08, ESCP-20 | Concurrencia masiva en consultas y publicaciones |
| ESC-02 | Disponibilidad | Fallo del servidor principal (administración, reseñas o perfiles) durante horario laboral de operación crítica | El sistema reinicia automáticamente el servicio o notifica al administrador para recuperación manual en tiempo de indisponibilidad <10 minutos | ESCP-02, ESCP-19, ESCP-26 | Failover automático de servicios críticos |
| ESC-03 | Usabilidad | Administrador sin experiencia técnica necesita cambiar estados de cuenta, configurar categorías, buscar reseñas, ordenar datos, editar perfil o actualizar foto desde móvil | El usuario completa todas las acciones sin asistencia externa mediante interfaz intuitiva con tutoriales y paneles amigables en <5 pasos y <3 minutos | ESCP-03, ESCP-05, ESCP-16, ESCP-21, ESCP-23, ESCP-28 | Interfaz intuitiva y consistente en todos los módulos |
| ESC-04 | Rendimiento | Cientos de usuarios malintencionados o administradores concurrentes solicitan simultáneamente reportes de cuentas de usuarios | El sistema activa limitación de peticiones (rate-limiting) o encola solicitudes para evitar saturación, manteniendo servicio disponible y rechazando excedentes en <1s | ESCP-04 | Procesamiento de volúmenes con protección DDoS |
| ESC-05 | Seguridad | Usuarios no autorizados intentan acceder al panel administrativo, editar reseñas ajenas, acceder a perfiles privados o subir archivos con extensiones no permitidas (.exe) | El sistema bloquea todos los intentos inmediatamente, registra eventos en log de auditoría (<1s), notifica al administrador y muestra mensaje de error sin comprometer seguridad | ESCP-06, ESCP-18, ESCP-25, ESCP-30 | Detección de amenazas y validación de acceso |
| ESC-06 | Mantenibilidad | Equipo de desarrollo solicita cambios en lógica de validación de usuarios, agregar filtros de reseñas, corregir validación de texto o agregar campo opcional al perfil | El cambio se implementa y despliega sin afectar otras funcionalidades del sistema mediante backend monolítico con componentes modulares en tiempo <4 horas con pruebas automatizadas | ESCP-07, ESCP-15, ESCP-22, ESCP-27 | Actualizaciones sin interrupción de servicio |
| ESC-07 | Confidencialidad | Usuario sin permisos intenta leer datos privados de perfil de otros usuarios mediante acceso directo a endpoints | El sistema enforza control estricto por rol/propiedad de recurso con 100% de endpoints verificando permisos antes de retornar datos | ESCP-10 | Protección de información privada |
| ESC-08 | Integridad | Atacante modifica token de acceso con firma inválida o realiza acción crítica desde sesión secuestrada en dispositivo desconocido | El sistema rechaza la solicitud inmediatamente, solicita re-autenticación o 2FA, invalida tokens previos y detecta cambio de IP en <3s con cierre de sesión global en <6s | ESCP-11, ESCP-12 | Validación de integridad de sesiones |
| ESC-09 | Disponibilidad | Pico de 100 logins por minuto de usuarios legítimos durante hora pico con verificación 2FA | El sistema escala horizontalmente con rate-limit por IP/usuario, procesando logins en <2s con error rate <1% y sin caída del servicio 2FA | ESCP-13 | Escalabilidad horizontal en autenticación |
| ESC-10 | Disponibilidad | Caída del proveedor externo de correo/OTP durante intermitencia del servicio de recuperación de contraseña | El sistema ejecuta reintentos con backoff exponencial y activa ruta alterna con códigos de respaldo, logrando éxito ≥99% en 15 minutos con notificación de estado <2 minutos | ESCP-14 | Continuidad operacional con servicios externos |
| ESC-11 | Rendimiento | Sistema automático recalcula calificación promedio de profesor tras recibir nueva calificación en tiempo real | El sistema recalcula el promedio mediante algoritmo optimizado y actualiza la base de datos en <100ms sin bloquear otras operaciones | ESCP-17 | Optimización de cálculos algorítmicos |
| ESC-12 | Rendimiento | Más de 100 usuarios concurrentes actualizan simultáneamente datos de sus perfiles personales durante horario pico | El sistema gestiona todas las solicitudes mediante procesamiento asíncrono sin colapsar operaciones, guardando cambios en <1s con tiempo medio de respuesta <2s bajo carga | ESCP-24, ESCP-29 | Procesamiento concurrente optimizado |
| ESC-13 | Seguridad | Usuario malintencionado intenta subir archivo con extensión no permitida (.exe, .bat, .sh) como foto de perfil desde navegador o app móvil | El sistema valida extensión y tipo MIME, rechaza archivo inmediatamente con 100% de efectividad y muestra mensaje de error claro sin comprometer seguridad del servidor | ESCP-30 | Validación y sanitización de entradas |
| ESC-14 | Usabilidad | Estudiante busca reseñas de un profesor específico mediante filtros avanzados y opciones de búsqueda | El sistema muestra reseñas ordenadas con filtros activos, paginación eficiente y resultados en <2 segundos con interfaz responsive | ESCP-16 | Mejora de experiencia de búsqueda |
| ESC-15 | Usabilidad | Usuario desea ordenar reseñas por diferentes criterios (fecha, calificación, relevancia) | El sistema muestra opciones de ordenamiento visibles y accesibles, permitiendo que el usuario encuentre la opción en <5 segundos con feedback inmediato | ESCP-21 | Personalización de interfaz |
| ESC-16 | Mantenibilidad | Desarrollador necesita agregar nueva funcionalidad de filtros avanzados al módulo de reseñas y calificaciones | El sistema permite implementar nuevos filtros mediante arquitectura de plugins sin modificar código existente en tiempo <4 horas con tests automatizados | ESCP-15 | Arquitectura modular y extensible |
| ESC-17 | Rendimiento | Administrador consulta listado general de usuarios con todos sus datos al iniciar el sistema durante jornada laboral | El sistema optimiza carga inicial mediante caché y consultas eficientes, mostrando información completa sin retrasos perceptibles en <5s con interfaz estable | ESCP-08 | Optimización de carga inicial |
| ESC-18 | Disponibilidad | Falla del servidor backend que afecta múltiples componentes (administración, reseñas, perfiles) del sistema monolítico | El sistema detecta la falla y restaura el proceso automáticamente mediante monitor de procesos (PM2/Supervisor) con healthchecks y restart policies en tiempo <30 segundos | ESCP-02, ESCP-19, ESCP-26 | Recuperación automática del servidor |
| ESC-19 | Usabilidad | Usuario autenticado desea editar información personal o actualizar foto de perfil desde dispositivo móvil o web | El sistema muestra formulario de edición con datos actuales precargados y permite previsualizar cambios antes de guardar en proceso de <2 pasos completado en <2 segundos | ESCP-23, ESCP-28 | Formularios precargados y responsivos |
| ESC-20 | Confidencialidad | Atacante intenta login con credenciales comprometidas o filtradas de bases de datos externas | El sistema rechaza autenticación sin revelar si el correo existe en la base de datos, registra el intento sospechoso y retorna respuesta HTTP 401 en <300ms | ESCP-09 | Comunicación segura de errores de autenticación |
